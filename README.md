### 1. TDD
- TDD란?
    - 테스트 주도 개발 (Test-Driven Development)
    - 테스트 코드를 먼저 작성
        1. 항상 실패하는 테스트를 먼저 작성하고 (Red)
        2. 테스트가 통과하는 프로덕션 코드를 작성하고 (Green)
        3. 테스트가 통과하면 프로덕션 코드를 리팩토링합니다. (Refactor)


- TDD를 공부해야 되는 이유
    - 대부분의 서비스 회사가 테스트 코드에 관해 요구

      [우대사항] - 빌드/테스트/배포 자동화 경험이 있으신분
    - 빠른 피드백
    - 자동 검증
    - 개발자가 만든 기능을 안전하게 보호
